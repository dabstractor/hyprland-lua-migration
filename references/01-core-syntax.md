# Reference 01 — Core Syntax & File Mechanics

> Companion to `SKILL.md`. Read this when you need the exhaustive mapping of
> top-level config mechanics: file location, comments, variables, sourcing,
> env, autostart, executing, sections, and value types.

Since Hyprland **0.55**, the configuration language switched from the bespoke
"hyprlang" INI-like syntax to **Lua**. The global table `hl` is Hyprland's API.
Everything you used to write as a keyword/section is now a function call on `hl`
(or a key inside an `hl.config({...})` table).

---

## 1. File location & loading

| Concept | 0.54 (hyprlang) | 0.55 (Lua) |
| --- | --- | --- |
| Default path | `~/.config/hypr/hyprland.conf` | `~/.config/hypr/hyprland.lua` |
| Custom path | `hyprland --config FILE` | `hyprland --config FILE` / `hyprland -c FILE` (unchanged) |
| Env override | `HYPRLAND_CONFIG=FILE` | `HYPRLAND_CONFIG=FILE` (unchanged) |
| Autogenerated marker | line `autogenerated = 1` | line `hl.config({ autogenerated = true })` |
| Reload | autosave, or `hyprctl reload` | autosave, or `hyprctl reload` (`reload full-reset` recreates context) |

If both `hyprland.lua` and `hyprland.conf` exist, the Lua one wins. `hyprctl reload full-reset` recreates the entire config context (lets you switch lua↔hyprlang) — rarely needed.

## 2. Comments

| 0.54 | 0.55 |
| --- | --- |
| `# a comment` | `-- a comment` |
| `##` → literal `#` | `--[[ block comment ]]` |

Lua line comments are `--`; block comments are `--[[ ... ]]`.

## 3. User variables

Hyprlang had its own variable system (`$VAR`). In Lua you just use ordinary Lua
locals/globals.

```ini
# 0.54
$terminal = kitty
$mainMod  = SUPER
bind = $mainMod, Q, exec, $terminal
```
```lua
-- 0.55
local terminal = "kitty"
local mainMod  = "SUPER"
hl.bind(mainMod .. " + Q", hl.dsp.exec_cmd(terminal))
```

Notes:
- Use `local` to avoid polluting the global namespace.
- String concatenation is `..` (two dots), not space.
- Old trick `col.active_border = ff$MyRed1111` (splicing into a token) no longer
  applies — build the whole string in Lua: `"0xff" .. myRed .. "1111"`.

## 4. Sourcing / multiple files

`source = path` is replaced by Lua's `require`.

```ini
# 0.54
source = ~/.config/hypr/myColors.conf
```
```lua
-- 0.55
require("myColors")          -- ~/.config/hypr/myColors.lua
require("awesomeconf/keybinds")   -- subdir; '/' or '.' both work
require("awesomeconf.animation")
```

- Paths are relative to `hyprland.lua`. Use `/` (UNIX) or `.` (generic) as separators.
- **Each `require()` runs in its own protected Lua scope**, so an error in one
  file does NOT abort the others. This is the big reason to split your config.
- Plain top-level code in `hyprland.lua` itself is NOT protected from itself — a
  runtime error there aborts the rest of that file (but not other `require`d files).
- `require("nonexistent")` throws and kills the calling file. Wrap with `pcall` if you need to tolerate missing modules:
  ```lua
  local ok, mod = pcall(require, "maybe-missing")
  if ok then print("loaded", mod) else print("missing:", mod) end
  ```
- Advanced: Hyprland's protected `require` can be bypassed via the original
  function `__require` (e.g. `require = __require`) for third-party modules that
  break under scope isolation.

## 5. Environment variables

`env = NAME,VALUE` → `hl.env("NAME", "VALUE")`.

```ini
# 0.54
env  = XCURSOR_SIZE,24
envd = XCURSOR_SIZE,24        # also export to D-Bus/systemd
```
```lua
-- 0.55
hl.env("XCURSOR_SIZE", "24")
hl.env("XCURSOR_SIZE", "24", true)   -- 3rd arg true == old envd (D-Bus export)
```

- Like the old keyword, `hl.env` only fires **on launch**; it is not re-applied on reload.
- Do NOT add quotes around the value — pass the raw string. Reference existing
  vars with `os.getenv`: `hl.env("SSH_AUTH_SOCK", os.getenv("XDG_RUNTIME_DIR") .. "/ssh-agent.socket")`.

## 6. Executing programs / autostart

The `exec` and `exec-once` keywords are **gone**. Two replacements:

| Old intent | New equivalent |
| --- | --- |
| `exec-once = cmd` (run only on first launch) | `hl.on("hyprland.start", function() hl.exec_cmd("cmd") end)` |
| `exec = cmd` (run on every reload) | call `hl.exec_cmd("cmd")` directly at top level |

```ini
# 0.54
exec-once = waybar
exec-once = nm-applet
exec      = hyprpaper
```
```lua
-- 0.55  (run only on first launch)
hl.on("hyprland.start", function()
    hl.exec_cmd("waybar")
    hl.exec_cmd("nm-applet")
    -- "waybar & hyprpaper & firefox" style chaining is unnecessary:
    -- each hl.exec_cmd is already async; just call them separately.
end)
```

- `hl.exec_cmd(cmd)` runs `sh -c cmd` asynchronously (no `& disown` needed).
- `hl.exec_raw(cmd)` runs the command directly without a shell (old `execr`).
- Spawn on exit with `hl.on("hyprland.shutdown", function() ... end)`.

## 7. Sections → `hl.config`

Old `name { ... }` blocks become keys inside a Lua table passed to `hl.config`.
Sub-sections (e.g. `decoration.blur`) become nested tables.

```ini
# 0.54
general {
    gaps_in = 5
    layout  = dwindle
}
decoration {
    blur {
        size = 3
    }
}
```
```lua
-- 0.55
hl.config({
    general = {
        gaps_in = 5,
        layout  = "dwindle",
    },
    decoration = {
        blur = {
            size = 3,
        },
    },
})
```

Key facts:
- You may call `hl.config()` **multiple times**; each call merges/updates only
  the keys you pass. Splitting across files/functions is idiomatic.
- Keys use `:` in internal form (e.g. `decoration:blur:size`) but in Lua you use
  nested tables: `decoration.blur.size`.
- Unknown keys produce a config error (and are ignored). The error popup lists them.

## 8. Value type changes

| Type | 0.54 | 0.55 |
| --- | --- | --- |
| bool | `true`/`false`, `yes`/`no`, `on`/`off`, `0`/`1` | Lua `true` / `false` **only** |
| int | `5` | `5` |
| float | `0.5` | `0.5` |
| string | `dwindle` | `"dwindle"` (quoted) |
| vec2 | `20 20` (space-separated) | `{ 20, 20 }` (table) or `{x=20, y=20}` |
| color | `rgba(b3ff1aee)`, `0xeeb3ff1a` | `"rgba(b3ff1aee)"`, `0xeeb3ff1a`, or `"#fafc21"` |
| gradient | `rgba(a) rgba(b) 45deg` | `"rgba(a)"` or `{ colors = {"rgba(a)","rgba(b)"}, angle = 45 }` |
| modmask (MOD) | `SUPER_SHIFT` / `SUPER + SHIFT` | `"SUPER + SHIFT"` string (in binds) |

### Color formats (0.55)

1. Web hash: `"#fafc21"`, `"#ddd"`, `"#fa3d7bff"`
2. `rgba()`: `"rgba(b3ff1aee)"` or decimal `"rgba(179,255,26,0.933)"` (no spaces)
3. `rgb()`: `"rgb(b3ff1a)"` or `"rgb(179,255,26)"`
4. Legacy integer (ARGB order): `0xeeb3ff1a`

### Gradient

A single color is fine: `col.active_border = "rgba(33ccffee)"`.
Multi-stop with angle uses a table:
```lua
col = {
    active_border = { colors = {"rgba(33ccffee)", "rgba(00ff99ee)"}, angle = 45 },
    inactive_border = "rgba(595959aa)",
}
```

### css_gaps

`gaps_in`/`gaps_out`/`float_gaps` now accept either an integer (all sides) or a
table for per-side values:
```lua
general = { gaps_in = 5 }
-- or
general = { gaps_in = { top = 5, bottom = 5, left = 10, right = 10 } }
```

## 9. New autostart-style one-shot patterns

Because you have a real language now, common patterns are trivial:

```lua
-- Run something only the first time, then never again
hl.on("hyprland.start", function() hl.exec_cmd("dbus-update-activation-environment --all") end)

-- Conditional autostart
hl.on("hyprland.start", function()
    if os.execute("command -v waybar") then hl.exec_cmd("waybar") end
end)
```

## 10. Error behavior

- Fundamental Lua syntax error → Hyprland refuses to reload and shows an error popup.
- Runtime Lua error in a file → that file aborts; an error is shown.
- Runtime **Hyprland type** error (e.g. wrong type to `hl.*`) → execution
  continues, an error is shown.
- Async errors (e.g. inside a keybind callback) → a notification pops up.
- If your main file errors before binds load, you lose those binds. Hyprland
  gives emergency binds **SUPER+Q/R/M** (terminal/run/exit) and mentions this.

There are timeouts/limits that kill runaway scripts (infinite loops, reentrant
events) so a bad config can't lock up the session.

## 11. LSP / autocompletions (new, recommended)

Hyprland ships autogenerated LuaLS stubs (the full `hl.*` API with types). They
are installed to `/usr/share/hypr/stubs/`. Add to `.luarc.json`:
```json
{ "workspace": { "library": ["/usr/share/hypr/stubs"] } }
```
This gives you hover types, autocomplete, and signatures for every `hl.*` call —
including the return objects (`hl.get_active_window()` etc.).

## 12. Testing live: the REPL

`hyprctl repl` starts an interactive Lua session bound to the live compositor
state:
```sh
hyprctl repl 'hl.get_active_window().class'
hyprctl repl           # interactive; Ctrl+D to exit
```
`hyprctl eval 'LUA'` runs a Lua string and returns "ok" or the error.
`hyprctl dispatch 'LUA'` is shorthand for `eval 'hl.dispatch(...)'`.
